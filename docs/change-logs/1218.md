## Key decisions

- **Opus is the default for advisor and brainstorm tasks** unless the caller explicitly selects a different model.
- **Structured output is stored as raw JSON text** so downstream code can parse it without a second re-encoding step.
- **Brainstorm fan-out is idempotent**: existing child `.task` files are left untouched on reruns.
- **Child brainstorm tasks are written with `# schedule: manual`** so the scheduler will not auto-fire them.
- **Default-account fallback is runtime-only** and only applies to advisor / brainstorm tasks that omit `account`.

## Gotchas

- The CLI response is parsed in two layers: the outer JSON wrapper first, then the model’s `result` or `response` payload.
- Code fences are tolerated in the inner payload, but the stored `structured_output` remains the cleaned raw JSON string.
- Fan-out failures are soft warnings only; they do not flip the parent run out of `success`.
