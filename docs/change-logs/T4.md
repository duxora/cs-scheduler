# T4 — Promote Accounts to top-level Hub app

**Replaces T3's short-lived /scheduler/settings/* shell. Accounts are global to the Hub, so they get their own sidebar entry.**

## What changed

- `frontend/src/apps/accounts/App.tsx` (new) — single-route shell that renders the existing `AccountsPage` from the Scheduler app. Cross-app import is intentional for v1; a future ticket can hoist the page itself if Scheduler stops needing the type.
- `frontend/src/router.tsx` — `/accounts/*` registered as a lazy top-level route.
- `frontend/src/shared/apps.ts` — new HUB_APPS entry (icon 🔑) between Scheduler and KB.
- `frontend/src/apps/scheduler/App.tsx` — removed the Settings tab and the `settings/*` route subtree shipped in T3. `/scheduler/accounts` and `/scheduler/settings*` now redirect to `/accounts` so existing bookmarks keep working.
- `frontend/src/apps/scheduler/pages/settings/{SettingsLayout,AccountsTab,NotificationsTab,DefaultsTab,BudgetsTab}.tsx` (deleted) — superseded.
- `frontend/src/apps/scheduler/pages/TaskNewPage.tsx` — `+ New` link now points at `/accounts`.
- `frontend/src/apps/scheduler/pages/AccountsPage.tsx` — name-availability check now distinguishes `unreachable` from `idle`. Network/4xx/5xx on the check endpoint surfaces "unreachable — restart server?" in red and keeps the Next button disabled. Catches the failure mode that bit us this session (stale uvicorn, see knowledge-base entry).

## Key decisions

- **Cross-app import for v1.** `apps/accounts/App.tsx` re-exports `AccountsPage` from `apps/scheduler/pages/`. Moving the file would mean rewiring `useAccounts`, `lib/api.ts`, types, and the relativeTime util. Cheaper to leave them where they are; refactor later if Scheduler ever drops the Account type.
- **Both old paths redirect.** `/scheduler/accounts` and `/scheduler/settings*` both `Navigate` to `/accounts` rather than 404, so any link or bookmark from the T1–T3 era still works.
- **Backend untouched.** Account routes stay at `/scheduler/api/accounts/...` — the FE still hits them through the existing `BASE`. Backend path normalization can be a follow-up if it ever adds value.
- **Fail-loud over silent reset.** A 404 on `check-name` used to silently rewind to `idle` — no UI signal, Next stuck. Now it states "unreachable" so the operator knows whether to restart vs investigate the form.

## Gotchas

- Codex sandbox couldn't write `.git/index.lock`, so the five settings file deletions had to be staged with `git rm` from the orchestrator after the worker exited.
- The Settings stub directory is removed entirely; `frontend/src/apps/scheduler/pages/settings/` no longer exists.
