# T3 — Settings shell with sidebar

**Third of three settings-UI tickets. Pure UI; no backend changes.**

## What changed

- `frontend/src/apps/scheduler/pages/settings/SettingsLayout.tsx` (new) — flex layout with `w-44` left sidebar and `<Outlet/>` content panel. Sidebar is a `NavLink` list (Accounts / Notifications / Defaults / Budgets) styled to match the existing top-nav rhythm.
- `frontend/src/apps/scheduler/pages/settings/AccountsTab.tsx` (new) — re-exports the existing `AccountsPage` unchanged.
- `frontend/src/apps/scheduler/pages/settings/{Notifications,Defaults,Budgets}Tab.tsx` (new) — stub pages, "Coming soon" + a one-liner about what each tab will hold.
- `frontend/src/apps/scheduler/App.tsx` — Accounts top-nav link replaced by Settings; nested routes under `/scheduler/settings`; old `/scheduler/accounts` URL redirects via `<Navigate replace>`.
- `frontend/src/apps/scheduler/pages/TaskNewPage.tsx` — inline "+ New" account link points at the new path.

## Key decisions

- **Re-export wrapper, not a move**, for AccountsPage. Keeps the file's import paths stable (`useAccounts`, `lib/api.ts`, etc. all live next to `pages/`) so a future "expand AccountsPage" change only touches one file.
- **Stub tabs ship now** with a placeholder + scope sentence so a future ticket can drop a real component in without touching layout or routing. The shell is the load-bearing change; the content is hot-swappable.
- **Redirect from `/scheduler/accounts` instead of removing it** — preserves any existing browser bookmarks and external links to the old URL.
- **Settings tab activates on prefix match** (no `end` prop) so it stays highlighted while inside any sub-tab.

## Gotchas

- Sidebar width is `w-44` (176px) — narrower would crowd "Notifications", wider would steal viewport from the content area.
- Index route under `/scheduler/settings` redirects to `accounts` so the bare URL never lands on an empty panel.
