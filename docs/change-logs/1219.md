# 1219 — accounts CLI helper + auth-failure monitor

**Closes the two operational gaps left after Phase 5: setup ergonomics and idle-profile detection.**

## What changed

- `src/claude_scheduler/cli.py` — new `cs accounts {list,add,remove,set-default}` subcommands. `add --kind config_dir` mkdirs `~/.claude-profiles/<name>` and runs `claude /login` interactively before inserting the row. `add --kind api_key` validates the `keychain:*` / `op://*` reference shape.
- `src/claude_scheduler/core/notify.py` — added `_auth_failure_pattern()` plus `notify_auth_failure()`. Emits desktop + Telegram alert with the exact `CLAUDE_CONFIG_DIR=<dir> claude /login` command.
- `src/claude_scheduler/core/executor.py` — failure branch now scans stderr and, when an account is bound and matches `kind=config_dir`, fires the auth alert and tags the result dict with `auth_failure: True`.
- `src/claude_scheduler/core/orchestrator.py` — suppresses the generic error notification (and remediation flow) when `auth_failure` is set, so re-login is the only nudge the user sees.
- `tests/test_cli_accounts.py`, `tests/test_auth_monitor.py` (new) — behavioral coverage for both slices.

## Key decisions

- **Manual `claude /login` is the supported credential path**: the CLI just shells out to it with `CLAUDE_CONFIG_DIR=<profile>` set. No token extraction, no PKCE-driven OAuth — both are blocked by the 2026-04-04 ToS update and §10 of the Phase 1 research doc.
- **Profile dir defaults to `~/.claude-profiles/<name>`** (mode 0700 best-effort). Operator can override with `--config-dir`.
- **Login is a hard gate**: if `<config_dir>/.credentials.json` doesn't exist after `claude /login` returns, the account is not registered. `--no-login` skips the gate for offline/scripted setups.
- **Auth-failure detection is gated to `kind=config_dir`** — API keys don't expire on the same path. Patterns are intentionally specific (eight precise regexes); generic "error" / "failed" matches are excluded to avoid suppressing real failures.
- **6-hour cooldown per account**, sentinel files under `~/.claude-scheduler/auth-alert-cooldown/<account_id>`. Prevents an idle profile from spamming on every cron tick.
- **`auth_failure` shadows the generic error notification** so the operator doesn't get two alerts for the same root cause.

## Gotchas

- `notify_auth_failure` lazy-imports inside the executor failure branch to keep the success path off `notify.py`.
- The Telegram + desktop callouts are wrapped in try/except in the executor — a notifier outage must never raise back into the run loop.
- Removing an account does **not** delete the on-disk profile dir; the path is printed so the operator can `rm -rf` it manually if desired.
