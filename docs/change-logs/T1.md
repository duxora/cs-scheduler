# T1 — Add-account modal: copy-paste login + verify polling

**First of three settings-UI tickets, builds on the accounts CLI shipped in 1219.**

## What changed

- `apps/scheduler/routes.py` — two read-only setup helpers: `GET /api/accounts/check-name?name=…` (availability) and `GET /api/accounts/check?config_dir=…` (returns `dir_exists` / `has_credentials` / `expanded_path`). Both placed before the `/{account_id}` path-param route so they aren't shadowed.
- `frontend/src/apps/scheduler/pages/AccountsPage.tsx` — `kind=config_dir` flow is now 2-step: details (with debounced live name-availability check + auto-derived `~/.claude-profiles/<slug>` config_dir) → login & verify (copy-paste `claude /login` block + Verify button polling `/check` every 2s up to 60s, auto-saves on `has_credentials=true`). `kind=api_key` keeps the single-step path.
- `frontend/src/apps/scheduler/lib/api.ts` — `checkAccountName`, `checkAccountCredentials`.
- `frontend/src/apps/scheduler/types/index.ts` — `AccountNameCheck`, `AccountCredentialCheck`.
- `tests/test_routes_accounts_check.py` (new) — 8 cases: empty/taken/available names; non-existent / dir-only / credentialed paths; `~`-expansion.

## Key decisions

- **Browser cannot drive `claude /login` directly** — the OAuth flow needs an interactive terminal session. Page emits a copy-paste command instead, then polls until `.credentials.json` lands. Stays inside the supported first-party CLI surface (per the §10 re-research note).
- **Auto-derive `config_dir`** from the typed name (`~/.claude-profiles/<slug>`) until the user manually edits the field. Reduces friction for the common case; advanced users can still override.
- **Polling is 2s × 30 attempts, abortable on modal close.** 60s ceiling is enough for a normal browser-based OAuth roundtrip; longer waits suggest the user got distracted, fail-soft to a `try again` state.
- **Auto-save on credential detection** so the operator never has to click Save twice. Errors during the implicit save show in the existing `actionError` panel without losing modal state.

## Gotchas

- Read-only check endpoints — never mutate, never run subprocess. Anything that fails returns `{dir_exists: false, has_credentials: false}` rather than 5xx, so the polling UI doesn't have to special-case error states.
- `Path(...).expanduser()` only — does NOT resolve symlinks (`realpath`). Avoids leaking symlink targets through the `expanded_path` field.
