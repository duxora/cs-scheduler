## Key decisions

- Plaintext secrets never enter the UI. `api_key` accounts accept only `keychain:<service>` or `op://<vault>/<item>/<field>` refs. The hint copy says so explicitly.
- For config_dir accounts, the modal tells the user to run `CLAUDE_CONFIG_DIR=<dir> claude /login` themselves — the scheduler never sees the OAuth token.
- Empty account value on a task means "use default" — backend resolves at runtime, no UI special-casing.
